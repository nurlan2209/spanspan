# Фича: Группы для тренировок

## Идея в двух словах

Клиент при регистрации вводит дату рождения → система вычисляет возраст → тренер создаёт группу и указывает возрастной диапазон → клиент видит подходящие группы и записывается → тренер подтверждает набор.

---

## 1. Профиль клиента — новые поля

### При регистрации добавить:
- **Дата рождения** — для определения возраста

Возраст вычисляется автоматически из даты рождения.

---

## 2. Группы — что это такое

Группа = конкретная тренировка с расписанием и возрастным ограничением.

### Поля группы:
| Поле | Тип | Описание |
|------|-----|----------|
| Название | string | Название группы |
| Описание | string | Доп. информация |
| Тренер | user_id | Кто ведёт |
| Дата и время | datetime | Когда тренировка |
| Макс. участников | number | Лимит мест |
| Возраст от | number | Минимальный возраст (включительно) |
| Возраст до | number | Максимальный возраст (включительно) |
| Статус | enum | `recruiting` / `confirmed` / `cancelled` / `completed` |

---

## 3. Флоу клиента

```
Регистрация
  → вводит телефон, имя, пароль, дату рождения
  → система вычисляет текущий возраст

Раздел "Группы"
  → показываются группы со статусом `recruiting`
  → фильтр: только те, куда клиент подходит по возрасту
  → видно: название, тренер, дата/время, сколько мест осталось

Карточка группы
  → кнопка "Записаться"
  → после записи — статус "Ожидаю подтверждения"
  → если тренер подтвердил группу — статус меняется на "Подтверждено"
  → если отменена — статус "Отменено"

Экран "Мои записи"
  → список групп, на которые записан клиент + их статусы
```

---

## 4. Флоу тренера

```
Тренер создаёт группу
  → указывает: название, описание, дата/время, макс. участников
  → указывает возраст: от X до Y лет

Экран "Мои группы"
  → список созданных групп
  → для каждой — число записавшихся / лимит
  → кнопка "Посмотреть участников" → список клиентов (имя, возраст)
  → кнопка "Подтвердить набор" → статус группы → `confirmed`
  → кнопка "Отменить" → статус → `cancelled`
```

## 5. База данных

### Изменения в таблице `users`:
```sql
ALTER TABLE users ADD COLUMN birth_date DATE;
```

### Новая таблица `groups`:
```sql
CREATE TABLE groups (
  id SERIAL PRIMARY KEY,
  title VARCHAR(255) NOT NULL,
  description TEXT,
  trainer_id INTEGER REFERENCES users(id),
  scheduled_at TIMESTAMP NOT NULL,
  max_participants INTEGER NOT NULL DEFAULT 20,
  age_min INTEGER NOT NULL,
  age_max INTEGER NOT NULL,
  status VARCHAR(20) DEFAULT 'recruiting',
  created_at TIMESTAMP DEFAULT NOW()
);
```

### Новая таблица `group_enrollments`:
```sql
CREATE TABLE group_enrollments (
  id SERIAL PRIMARY KEY,
  group_id INTEGER REFERENCES groups(id) ON DELETE CASCADE,
  client_id INTEGER REFERENCES users(id),
  enrolled_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(group_id, client_id)
);
```

## 6. API эндпоинты

### Группы
| Метод | URL | Кто | Что делает |
|-------|-----|-----|------------|
| `GET` | `/api/groups` | Клиент | Список подходящих групп (фильтр по возрасту клиента) |
| `GET` | `/api/groups/trainer` | Тренер | Мои группы |
| `POST` | `/api/groups` | Тренер | Создать группу |
| `PATCH` | `/api/groups/:id/confirm` | Тренер | Подтвердить набор |
| `PATCH` | `/api/groups/:id/cancel` | Тренер | Отменить группу |

### Записи
| Метод | URL | Кто | Что делает |
|-------|-----|-----|------------|
| `POST` | `/api/groups/:id/enroll` | Клиент | Записаться |
| `DELETE` | `/api/groups/:id/enroll` | Клиент | Отписаться |
| `GET` | `/api/groups/:id/members` | Тренер | Список участников |
| `GET` | `/api/enrollments/my` | Клиент | Мои записи |
